# Taxonomie des Clauses Juridiques — Contrats de Développement Logiciel

## 1. Propriété Intellectuelle (PI)

### Sous-catégories
- **Cession des droits** : Transfert de propriété du code source et des livrables au client
- **Licence d'utilisation** : Droits accordés sans transfert de propriété
- **PI préexistante** : Composants, bibliothèques ou code existant avant le projet
- **PI développée** : Code et livrables créés spécifiquement pour le projet
- **Code open source** : Utilisation et obligations liées aux licences open source

### Points de vérification
- Qui détient les droits sur le code produit ?
- Les droits sur la PI préexistante sont-ils préservés ?
- Les obligations liées aux licences open source sont-elles respectées ?
- Le client a-t-il une licence perpétuelle ou limitée dans le temps ?

## 2. Confidentialité et Protection des Données

### Sous-catégories
- **NDA / Entente de confidentialité** : Obligations de non-divulgation
- **Données personnelles** : Conformité Loi 25 / LPRPDE / RGPD
- **Données du client** : Protection des données métier et commerciales
- **Durée de confidentialité** : Période pendant et après le contrat
- **Exceptions** : Information publique, développement indépendant

### Points de vérification
- La durée de confidentialité est-elle cohérente entre les documents ?
- Les obligations de protection des données personnelles sont-elles couvertes ?
- Les exceptions sont-elles les mêmes dans les deux documents ?

## 3. Responsabilité et Garanties

### Sous-catégories
- **Limitation de responsabilité** : Plafond de dommages (souvent limité au montant du contrat)
- **Exclusion de dommages** : Dommages indirects, perte de profits, etc.
- **Garantie de conformité** : Le livrable respecte les spécifications
- **Garantie de bon fonctionnement** : Période post-livraison (ex: 30, 60, 90 jours)
- **Garantie contre les vices** : Absence de bugs majeurs
- **SLA (niveaux de service)** : Temps de réponse, disponibilité

### Points de vérification
- Le plafond de responsabilité est-il le même ?
- La période de garantie est-elle cohérente ?
- Les exclusions de dommages sont-elles alignées ?
- Les SLA mentionnés dans l'offre correspondent-ils au contrat ?

## 4. Conditions Financières

### Sous-catégories
- **Tarification** : Taux horaire, forfait, mixte
- **Conditions de paiement** : Délais (net 30, net 45, etc.)
- **Facturation** : Fréquence, jalons, pourcentages
- **Dépenses** : Frais remboursables, plafonds
- **Révision des prix** : Indexation, conditions de changement
- **Pénalités de retard** : Intérêts, montants

### Points de vérification
- Les taux horaires correspondent-ils ?
- Les conditions de paiement sont-elles les mêmes ?
- Les jalons de facturation sont-ils cohérents avec le calendrier projet ?

## 5. Portée et Livrables

### Sous-catégories
- **Périmètre du projet** : Ce qui est inclus et exclu
- **Livrables** : Documents, code, environnements
- **Critères d'acceptation** : Comment les livrables sont validés
- **Gestion des changements** : Processus pour les demandes hors scope
- **Calendrier** : Jalons, dates clés, durée

### Points de vérification
- La portée décrite dans l'offre respecte-t-elle les limites du contrat cadre ?
- Les livrables listés sont-ils couverts par le contrat ?
- Le processus de gestion des changements est-il cohérent ?

## 6. Résiliation et Fin de Contrat

### Sous-catégories
- **Résiliation pour convenance** : Préavis requis, conditions
- **Résiliation pour cause** : Manquements, défaut de paiement
- **Obligations post-résiliation** : Transfert de connaissances, restitution
- **Pénalités de résiliation** : Frais, dédommagement

### Points de vérification
- Les conditions de résiliation sont-elles les mêmes ?
- Les obligations de transition sont-elles couvertes ?

## 7. Résolution des Litiges

### Sous-catégories
- **Juridiction** : Province/pays applicable
- **Loi applicable** : Droit civil du Québec, common law, etc.
- **Médiation/Arbitrage** : Mécanismes alternatifs
- **Escalade** : Processus interne avant litige formel

### Points de vérification
- La juridiction est-elle la même dans les deux documents ?
- Le mécanisme de résolution est-il cohérent ?

## 8. Assurances et Conformité

### Sous-catégories
- **Assurance responsabilité professionnelle** : Montants minimaux
- **Assurance erreurs et omissions** : Couverture E&O
- **Conformité réglementaire** : Normes sectorielles du client
- **Certifications** : ISO, SOC2, etc.

### Points de vérification
- Les montants d'assurance requis sont-ils respectés ?
- Les exigences de conformité sont-elles reflétées dans l'offre ?
